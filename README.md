# Парсер судебной практики Sudact.ru по делам о незаконном использовании товарного знака

## О проекте

В репозитории опубликован небольшой исследовательский проект, посвященный сбору, очистке и анализу судебных актов с сайта **Sudact.ru**.  
Основная цель — автоматизировать парсинг судебной практики по заданному текстовому запросу и затем извлечь из текстов решений ключевые данные для дальнейшего исследовательского анализа.

Проект демонстрирует полный цикл работы с судебными данными: от сбора на сайте Sudact.ru до очистки, анализа и интерпретации текстов судебных актов.  
Он показывает, что даже относительно небольшой и «грязный» массив судебных решений можно превратить в исследовательский датасет, пригодный для анализа судебной практики в конкретной правовой области.

Проект сфокусирован на делах о **незаконном использовании товарного знака** и использует решения:
- Арбитражного суда города Москвы (АСГМ),
- Арбитражного суда Новосибирской области (АС НСО).

## Что делает проект

### 1. Первичный парсинг
На вход подаются:
- наименование суда;
- целевой период;
- текстовый запрос.

Далее парсер имитирует действия пользователя на сайте Sudact.ru и извлекает:
- номер дела;
- дату судебного решения;
- ссылку на судебный акт;
- служебные параметры запроса.

Результат первого этапа сохраняется в промежуточный датасет.

### 2. Углубленный парсинг
На втором этапе проект переходит по ссылкам на тексты решений и извлекает из них:
- предмет иска;
- резолютивную часть;
- суммы, заявленные истцом;
- суммы, удовлетворенные судом.

Для этого используются регулярные выражения и текстовая обработка, так как структура судебных актов сильно варьируется и не всегда поддается простому шаблонному поиску.

### 3. Очистка и предобработка данных
После объединения данных выполняются:
- обработка пропусков;
- удаление дубликатов;
- удаление строк без значимой информации;
- фильтрация заведомо некорректных случаев;
- устранение явных выбросов, например ситуаций, когда удовлетворенная сумма больше заявленной.

### 4. Исследовательский анализ данных
В проекте проведен EDA по следующим направлениям:
- общая статистика;
- распределение заявленных и взысканных сумм;
- анализ данных по годам, месяцам и судам;
- корреляционный анализ количественных признаков.

### 5. Анализ значимости слов
Для текстов судебных решений применяется **TF-IDF**, чтобы оценить, какие слова наиболее характерны для собранной выборки и насколько она соответствует целевому запросу пользователя.
